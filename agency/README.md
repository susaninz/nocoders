# 🏢 Agency — Система мультиагентной разработки

> База агентов для вайбкодинга. Нанимай команду под проект.

## 🎯 Концепция

**Agency** — это библиотека готовых AI-агентов (промптов) для Cursor.
Каждый агент имеет свою специализацию, характер и зону ответственности.

```
┌─────────────────────────────────────────────────────────────────┐
│                         🏢 AGENCY                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ШТАТНЫЕ (staff/) — всегда в команде:                           │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐           │
│  │🤝 НАПАРНИК│ │🛠️ РАЗРАБ │ │📜 ЛЕТОПИС│ │📢 ГЛАШАТАЙ│           │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘           │
│                                                                  │
│  ФРИЛАНСЕРЫ (freelance/) — нанимаем по необходимости:           │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐           │
│  │🔍 РЕВИЗОР │ │🔧 МЕХАНИК│ │🎨 SANDBOX│ │✍️ EDITOR │           │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘           │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 📋 Штатные агенты (staff/)

Базовая команда для любого проекта.

| Агент | Файл | Роль | Когда использовать |
|-------|------|------|-------------------|
| 🤝 **НАПАРНИК** | `напарник.md` | Идеи, архитектура, обучение | "Давай обсудим...", "Объясни..." |
| 🛠️ **РАЗРАБ** | `разраб.md` | Код, фичи, баги | "Реализуй...", "Исправь..." |
| 📜 **ЛЕТОПИСЕЦ** | `летописец.md` | База знаний, документация | "Запиши...", "Добавь в базу..." |
| 📢 **ГЛАШАТАЙ** | `глашатай.md` | Публикации, Telegram канал | "Опубликуй...", "Сделай пост..." |

---

## 🎨 Фрилансеры (freelance/)

Специалисты, которых нанимаем под задачу.

| Агент | Файл | Роль | Когда нанимать |
|-------|------|------|----------------|
| 🔍 **РЕВИЗОР** | `ревизор.md` | Code review, рефакторинг | Сложный код, нужна проверка |
| 🔧 **МЕХАНИК** | `механик.md` | Деплой, инфраструктура | Есть prod, нужен DevOps |
| 🎨 **SANDBOX** | `sandbox.md` | Прототипы, эксперименты | Фронтенд, тестовые страницы |
| ✍️ **EDITOR** | `editor.md` | Тексты, копирайтинг | Контент для сайта |
| 🧪 **ТЕСТЕР** | `тестер.md` | Тесты, QA | Критичный код, нужны тесты |
| 📊 **АНАЛИТИК** | `аналитик.md` | Метрики, данные | Нужна аналитика |

## 🚀 Как использовать

### 1. Новый проект — запусти онбординг

Открой `_onboarding.md` и обсуди с НАПАРНИКОМ:

- Что за проект?
- Какой стек?
- Кого берём в команду?

### 2. Копируй промпты в проект

```bash
# Структура проекта
project/
├── .cursor/rules/
│   ├── напарник.md      # ← копируем из agency/staff/
│   ├── разраб.md
│   ├── ревизор.md       # ← копируем из agency/freelance/
│   └── ...
└── .agents/
    ├── напарник/        # Рабочее пространство агента
    ├── разраб/
    └── ...
```

### 3. Универсальные агенты — через symlink

ЛЕТОПИСЕЦ и ГЛАШАТАЙ работают со всеми проектами:

```bash
# Создаём ссылки на универсальных агентов
ln -s ~/Cursor/nocoders/.agents/летописец project/.agents/летописец
ln -s ~/Cursor/nocoders/.agents/глашатай project/.agents/глашатай
```

## ➕ Добавить нового агента

1. Создай файл в `freelance/` (или `staff/` если базовый)
2. Опиши: роль, характер, задачи, примеры использования
3. Закоммить в nocoders
4. Используй в проектах

## 💡 Принципы

1. **Один агент — одна задача** — не смешивай контексты
2. **Передавай эстафету** — закончил этап → иди к следующему
3. **Штат + фриланс** — базовая команда + специалисты по необходимости
4. **Git = версионирование** — история в коммитах, не в файлах
